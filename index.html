Levis Parking

During my second year I was assigned to a project team where we worked in a team with our classmates to create a specific app for a client. Our app was meant to make it easy to find/reserve parking spaces in Saskatoon, it would be made for our client, Levis Tech, and it would be named “Levis Parking”.

This project challenged me and my group in many ways, the first and most significant being getting off the ground. We were working with a technology very unfamiliar to us at the time, called React Native. React Native is a JavaScript library that allows you to create apps for Android and iOS using React. For the first month of our project, we made almost no progress as we could not get the technology working correctly. We eventually had to restart the entire project from scratch, using a new file layout and prioritizing a clean project, which solved the issue. This first hurdle taught me not to get too attached to my work, as all it took was a few mistakes for us to have to completely restart our project.

We eventually found a rhythm and managed to deliver a product that we were all proud of. I was largely responsible for implementing the vehicle portion of our app. I created a way for our users to register their vehicles into the app, and then select which vehicle they would use to park. We got to showcase our app in a presentation in front of the industry’s leading programmers, and to all our classmates. CrowdStrike Bug

I also got firsthand experience with a disaster scenario, being the CrowdStrike bug of 2024. A security software known as “CrowdStrike” released a faulty software update that caused a worldwide outage on millions of computers, including Nutrien ones.

Multiple blue screens of death caused by a faulty software update on baggage carousels at LaGuardia Airport, New York City.
Via Wikipedia

I was called in at 3:00AM on my day off to assist in restoring operations to the site. Around 70% of the site’s computers were down, and we had to manually go to every single one and turn them back on. Once we found a solution it wasn’t hard to revert the update, but the hard part was tracking down all the important computers. By 11:00AM we had restored access to all the high priority machines, but it would have been another month before we had completely restored access to all Nutrien’s end users.

Though I do not have any physical evidence of such an event, it is unmistakable that I was present for the recovery process. And I feel I learned much more in this disaster event than I could ever learn from a project artifact.

Home Server

To streamline my digital life and expand my technical skills, I have built a home server that serves multiple purposes.

It hosts a centralized file system I can use to store any important files I need, I can also send more files to it, and access said files remotely with the correct credentials.

I’ve also configured several multiplayer game servers that I use to play games with my friends. This allows me to control the traffic that goes into the server, and so we don’t have to pay third parties for their services.

A Minecraft server I run using Fork, a program to simplify running a server by using a GUI

Additionally, the server functions as a reliable backup solution for all my sensitive data and schoolwork. My laptop, home computers, and phone will periodically (once a week) send the data I specify to my server. Where it will be kept safe and ready to deploy in case my devices become corrupted.

My home server has served as a fun passion project, a helpful tool, and an excellent way to refine my IT skills. I hope to work on it for many more years and add more features as I learn about them.
